# tritium-bridge

Local-first agent runtime. Picks an agent, generates an action (journal entry,
message-board post, or email) using their persona via LM Studio, and writes it
back into the shared `.tritium` world tree. Polls Gmail for replies from
Scotty and routes them to the right agent's inbox.

This service is the heartbeat that keeps the Tritium team "alive" while
Scotty is offline.

## Stack

- Python 3.11+ (tested on 3.12)
- `httpx` for the LM Studio OpenAI-compatible HTTP client
- `python-dotenv` for env loading
- stdlib `smtplib` + `imaplib` for mail
- Windows Task Scheduler as the cron driver

## Quick start

```powershell
# 1. Install deps
python -m pip install --user -r requirements.txt

# 2. Copy env, fill creds
Copy-Item .env.example .env
notepad .env

# 3. Smoke test
python -m tritium_bridge --check
python -m tritium_bridge --tick --dry-run --agent Bridge --action journal

# 4. Real run (writes to world)
python -m tritium_bridge --tick --agent Sol --action journal

# 5. Register the scheduled task
.\scripts\install-task.ps1
```

## Modes

| Flag | Behavior |
| ---- | -------- |
| `--check` | Validate env, persona files, and LM Studio reachability. |
| `--tick` | Run one action and exit. Picks agent + action randomly unless overridden. |
| `--tick --agent <Name> --action <kind>` | Force a specific agent and action. |
| `--dry-run` | Generate but do not send/write. |
| `--imap-watch` | Continuous foreground IMAP poll. Optional. |

`--tick` also polls IMAP once per invocation, so the scheduled task alone is
enough to receive Scotty's replies.

## Honesty footer

Every team-authored email includes a footer disclosing the LLM, the model,
and that it was generated by the agent's persona. This is non-negotiable.

## Layout

```
tritium-bridge/
  tritium_bridge/      # package
  scripts/             # task scheduler helpers
  sent/                # archived outgoing emails
  outbox/              # SMTP-failure fallback
  logs/                # bridge.log lives here
```
